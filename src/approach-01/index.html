Approach 01

Role-Based Semantic Tokens

Three-layer token architecture: primitive palette → semantic role → component alias. Components consume only the component-level aliases (--c-*), which are resolved per color role via data-role. No raw values inside component selectors.

Token Architecture

--primitive-gold-600: #956b38 → --role-primary-base: var(--primitive-gold-600) → --c-base: var(--role-primary-base)
--primitive-gold-300: #e8cc94 → --role-primary-border: var(--primitive-gold-300) → --c-border: var(--role-primary-border)
--primitive-gold-100: #fdf7ed → --role-primary-subtle: var(--primitive-gold-100) → --c-subtle: var(--role-primary-subtle)
