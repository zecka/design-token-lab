Approach 01

Role-Based Semantic Tokens

Three-layer token architecture: primitive palette → semantic role → component alias. Components consume only the component-level aliases (--c-*), which are resolved per color role via data-role. No raw values inside component selectors. Multi-theming POC: only the semantic role layer is re-mapped per data-theme × data-mode — the component layer never changes.

Token Architecture

Facet Surface role Intent
--role-*-subtle tinted background Very light wash of the role's hue. Used as the fill for soft badges and as the hover background for outline badges. Never strong enough to carry text alone.
--role-*-border edge / boundary line Mid-tone that defines the boundary of a surface without dominating it. Also serves as the hover fill for soft badges, stepping up from subtle.
--role-*-text ink on a light surface Darker shade readable against subtle or transparent backgrounds. Used for the label in soft and outline variants. Must pass WCAG AA against --role-*-subtle.
--role-*-base solid fill The role's full-strength color. Fills the highlight variant entirely — the most assertive expression of the role.
--role-*-base-hover solid fill on interaction Slightly darker (or lighter in dark mode) variant of base. Applied on hover and active states for the highlight badge.
--role-*-on-base ink on a solid fill Text color placed directly on top of base. Must achieve WCAG AA contrast (≥ 4.5 : 1) against --role-*-base.
--primitive-gold-600: #956b38 → --role-primary-base: var(--primitive-gold-600) → --c-base: var(--role-primary-base)
--primitive-gold-300: #e8cc94 → --role-primary-border: var(--primitive-gold-300) → --c-border: var(--role-primary-border)
--primitive-gold-100: #fdf7ed → --role-primary-subtle: var(--primitive-gold-100) → --c-subtle: var(--role-primary-subtle)
