CSS Design Token Exploration

Semantic Color Token Approaches

A playground for comparing different strategies for naming, structuring, and mapping CSS custom property color tokens. Each approach is fully standalone and can be evaluated independently.

  • 01 Role-Based Semantic Tokens primitive → role → component · Badge / Tag component →
  • 02 Facet Contract Tokens role facets → badge slots · consistent variant behavior →
  • 03 Chromatic Dimension Tokens chroma · echo · trace · mark · script — tokens named after perceptual color qualities →

Token Principles

  • Components never reference raw color values directly inside component selectors.
  • Token names are theme-agnostic — no "light" or "dark" in any name.
  • Color roles describe intent (primary, success, error) not appearance.
  • The number of layers between raw values and components varies per approach.
  • Each approach folder is self-contained: one HTML file, one CSS file.
