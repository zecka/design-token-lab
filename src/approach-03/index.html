Approach 03

Chromatic Dimension Tokens

Tokens are named after perceptual chromatic qualities — chroma, echo, trace, mark, script — rather than component surfaces or semantic intent. Each role binds its five dimensions once; variant expressions compose freely from those slots without knowing the underlying hues.

Token Architecture

Token Role
--chroma-*vivid, saturated fill
--echo-*faint tinted background
--trace-*border / edge line
--mark-*ink on a vivid surface
--script-*ink on a soft/transparent surface
--chroma-primary: #9B7B2E → --b-chroma: var(--chroma-primary) → background-color: var(--b-chroma)
--echo-success: #EAF2EC → --b-echo: var(--echo-success) → background-color: var(--b-echo)
--trace-evening: #7B6A9B → --b-trace: var(--trace-evening) → border-color: var(--b-trace)
--mark-primary: #FDFAF2 → --b-mark: var(--mark-primary) → color: var(--b-mark)
--script-error: #6A3324 → --b-script: var(--script-error) → color: var(--b-script)
