Approach 04

Contextual Slot Binding

Palette tokens (--pal-*) are declared per [data-theme][data-mode] scope. Each [data-role] element binds palette values into a compact set of local rendering slots — --_bg, --_fg, --_tint-bg, --_tint-fg, --_stroke. Component rules consume only those slots, remaining completely oblivious to which role or theme is active.

Token Architecture

[data-theme][data-mode] --pal-primary-solid: #B07D3E → [data-role="primary"] --_bg: var(--pal-primary-solid) → .badge[highlight] background: var(--_bg)
[data-theme][data-mode] --pal-primary-tint: #F5E4C8 → [data-role="primary"] --_tint-bg: var(--pal-primary-tint) → .badge[soft] background: var(--_tint-bg)
[data-theme][data-mode] --pal-primary-solid: #B07D3E → [data-role="primary"] --_stroke: var(--pal-primary-solid) → .badge[outline] border-color: var(--_stroke)
[data-theme][data-mode] --_card-border-hover: var(--pal-primary-solid) → .card:hover border-color: var(--_card-border-hover)
